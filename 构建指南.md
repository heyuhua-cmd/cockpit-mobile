## 1. 项目简介

Cockpit 是由 Blue Robotics 开发的**跨平台无人载具地面控制站**（Ground Control Station），支持：

- 水下 ROV（ArduSub）
- 水面无人船（ArduRover）
- 无人机（ArduCopter / ArduPlane）
- 地面车辆

**运行形态：**

| 形态 | 说明 |
|------|------|
| 浏览器（Lite 版） | 功能受限，无需安装 |
| **桌面 App（完整版）** | 基于 Electron，功能完整，推荐 |
| Docker | 用于服务端部署 |
| BlueOS 扩展 | 嵌入式平台使用 |

本文档主要介绍如何将源码构建为**桌面 App（.exe / .dmg / .AppImage）**。

## 2. 环境准备

### 2.1 必需工具

| 工具 | 最低版本 | 说明 | 下载地址 |
|------|----------|------|----------|
| **Node.js** | 18.x LTS | JavaScript 运行环境 | https://nodejs.org/ |
| **Yarn** | 1.x 或 4.x | 包管理器 | `npm install -g yarn` |
| **Git** | 2.x | 代码管理，打包时需要读取版本号 | https://git-scm.com/ |
| **Python** | 3.x | 编译原生 Node 模块 | https://www.python.org/ |

### 2.2 Windows 额外依赖

Windows 上需要安装 **C++ 构建工具**，用于编译 `serialport`、`@kmamal/sdl` 等原生模块。

**方法一（推荐）：安装 windows-build-tools**
```powershell
# 以管理员身份运行 PowerShell
npm install -g windows-build-tools
```

**方法二：手动安装 Visual Studio Build Tools**
1. 前往 https://visualstudio.microsoft.com/downloads/
2. 下载"生成工具 for Visual Studio"
3. 安装时勾选 **"C++ 桌面开发"** 工作负载

### 2.3 验证环境

打开终端（PowerShell / Terminal），依次运行：

```bash
node --version    # 应显示 v18.x.x 或以上
yarn --version    # 应显示版本号
git --version     # 应显示版本号
python --version  # 应显示 3.x.x
```

## 3. 获取源码

### 方法 A：已有源码（你当前的情况）

你已有源码位于 `D:/cockpit-master (2)/cockpit-master/`，直接进入该目录：

```powershell
cd "D:/cockpit-master (2)/cockpit-master"
```

**初始化 Git（若尚未初始化，打包版本号需要）：**

```powershell
git init
git add .
git commit -m "init"
git tag v1.0.0   # 创建版本标签，否则打包命令会报错
```

### 方法 B：从 GitHub 克隆（含子模块）

```bash
git clone --recurse-submodules https://github.com/bluerobotics/cockpit.git
cd cockpit
```

> ⚠️ 必须加 `--recurse-submodules`，否则子模块不会下载，编译会失败。

## 4. 安装依赖

```powershell
cd "D:/cockpit-master (2)/cockpit-master"
yarn install
```

### 安装过程说明

安装完成后会自动执行 `postinstall` 脚本，它会：

1. **下载 FFmpeg 二进制**（`scripts/download-ffmpeg.js`）  
   用于视频录制功能，文件较大（约 100MB），需要访问 GitHub，确保网络畅通
   
2. **下载 go2rtc 二进制**（`scripts/download-go2rtc.js`）  
   用于 WebRTC 视频流，同样需要从 GitHub 下载

3. **验证 SDL 库**（`scripts/verify-sdl.js`）  
   用于手柄（Joystick）支持

> 💡 **网络问题？** 如果 GitHub 下载缓慢，可设置代理：
> ```powershell
> $env:HTTPS_PROXY = "http://127.0.0.1:7890"   # 替换为你的代理地址
> yarn install
> ```

## 5. 开发模式运行

**在正式打包前，建议先用开发模式验证能否正常运行。**

### 5.1 浏览器模式（Lite 版，仅前端）

```powershell
yarn dev
```

启动后在浏览器访问：`http://localhost:5173`  
（功能受限，不含 Electron 特性）

### 5.2 Electron 桌面模式（完整版）⭐推荐

```powershell
yarn dev:electron
```

会自动启动 Electron 窗口，支持热重载（修改代码后自动刷新）。

## 6. 打包为桌面应用

### 6.1 Windows 安装包（.exe）

```powershell
# 确保已设置版本标签
git tag v1.0.0

# 打包（生成 NSIS 安装程序）
yarn deploy:electron:windows
```

打包完成后，安装包位于：
```
dist/
└── Cockpit-win-x64-1.0.0.exe    ← 安装程序（可分发）
└── win-unpacked/                 ← 解压版（免安装直接运行）
```

### 6.2 macOS 安装包（.dmg）

```bash
# Intel Mac
yarn deploy:electron:mac:x64

# Apple Silicon (M1/M2/M3)
yarn deploy:electron:mac:arm64
```

> ⚠️ macOS 需要 Apple 开发者证书才能通过 Gatekeeper。  
> PR 构建版本（未签名）绕过方法：
> ```bash
> xattr -d com.apple.quarantine /Applications/Cockpit.app
> ```

### 6.3 Linux（.AppImage / Flatpak）

```bash
# AppImage（通用 Linux）
yarn deploy:electron

# Flatpak（Steam OS / 受限环境）
yarn deploy:flatpak
```

### 6.4 仅构建，不打包安装程序（快速验证）

```powershell
yarn deploy:electron:dir
```

生成解压目录（不生成安装包），速度更快，适合测试。

### 6.5 打包产物说明

| 文件 | 说明 |
|------|------|
| `Cockpit-win-x64-*.exe` | Windows 安装程序（NSIS，支持自定义安装路径）|
| `Cockpit-mac-x64-*.dmg` | macOS Intel 磁盘镜像 |
| `Cockpit-mac-arm64-*.dmg` | macOS Apple Silicon 磁盘镜像 |
| `Cockpit-linux-x64-*.AppImage` | Linux 通用包（免安装）|

## 7. 常见问题排查

### ❌ 错误：`git describe --tags` 失败

**症状：** 打包时报错 `fatal: No names found, cannot describe anything`

**原因：** 仓库没有 git tag

**解决：**
```powershell
git init          # 如果还没有 git 仓库
git add .
git commit -m "init"
git tag v1.0.0
```

---

### ❌ 错误：原生模块编译失败（node-gyp）

**症状：** `yarn install` 时报 `gyp ERR!` 或 `MSBUILD: error`

**解决（Windows）：**
```powershell
# 以管理员身份运行
npm install -g node-gyp
npm install -g windows-build-tools

# 或者安装 Visual Studio Build Tools 后重试
yarn install
```

---

### ❌ 错误：FFmpeg / go2rtc 下载失败

**症状：** `Error: Failed to download ffmpeg` 或网络超时

**解决方案 1：设置代理**
```powershell
$env:HTTPS_PROXY = "http://127.0.0.1:7890"
$env:HTTP_PROXY  = "http://127.0.0.1:7890"
yarn install
```

**解决方案 2：手动下载**
1. 前往 https://github.com/BtbN/FFmpeg-Builds/releases 下载 FFmpeg Windows 版
2. 解压后将 `ffmpeg.exe` 放入 `binaries/ffmpeg/` 目录
3. 前往 https://github.com/AlexxIT/go2rtc/releases 下载对应平台的 go2rtc
4. 放入 `binaries/go2rtc/` 目录
5. 再次运行 `yarn install`（跳过下载）

---

### ❌ 错误：Electron 版本不兼容

**症状：** `The module was compiled against a different Node.js version`

**解决：**
```powershell
# 重新编译原生模块以适配 Electron 的 Node 版本
yarn add -D @electron/rebuild
npx electron-rebuild
```

---

### ❌ 错误：`Cannot find module` / 子模块缺失

**症状：** 运行时提示某个模块找不到

**原因：** 如果是从 GitHub clone 的但没带 `--recurse-submodules`

**解决：**
```bash
git submodule update --init --recursive
```

---

### ❌ 打包后 App 闪退或白屏

**排查步骤：**
1. 先用 `yarn dev:electron` 确认开发模式能正常运行
2. 打包时加上 `--dir` 参数（不打成安装包）直接运行可执行文件看报错
3. 查看日志文件：
   - **Windows：** `%APPDATA%\Cockpit\logs\`
   - **macOS：** `~/Library/Logs/Cockpit/`
   - **Linux：** `~/.config/Cockpit/logs/`

## 8. 附录：各平台注意事项

### Windows

- 建议使用 **PowerShell 5.1+** 或 **Windows Terminal**
- 如果路径含有空格（如 `cockpit-master (2)`），操作时需要用引号包裹路径
- 安装包类型为 **NSIS**，支持静默安装：
  ```powershell
  Cockpit-win-x64-1.0.0.exe /S   # 静默安装到默认路径
  ```
- 已安装的应用数据存储在 `%APPDATA%\Cockpit\`

### macOS

- 需要 Xcode Command Line Tools：`xcode-select --install`
- 商业分发需要 Apple Developer 账号进行签名和公证（Notarization）
- 应用数据存储在 `~/Library/Application Support/Cockpit/`

### Linux

- 推荐使用 Ubuntu 20.04+ / Debian 11+ 进行构建
- 需要安装 `libsdl2-dev`（手柄支持）：
  ```bash
  sudo apt install libsdl2-dev
  ```
- AppImage 运行时需要给可执行权限：
  ```bash
  chmod +x Cockpit-linux-x64-1.0.0.AppImage
  ./Cockpit-linux-x64-1.0.0.AppImage
  ```
- 应用数据存储在 `~/.config/Cockpit/`
